# XHTML 뷰어 시스템 — 아키텍처·구현·로드맵

## 1. 문서 구조 및 렌더링

### 1.1 블록 기반 XHTML

- **combined.xhtml**: 도서 전체를 단일 XHTML로 통합.
- **블록 앵커**: `data-chapter-index`, `data-block-index`로 위치 추적.

### 1.2 렌더 방식 (구현 확정)

| 항목 | 구현 |
|------|------|
| **iframe** | **사용하지 않음** |
| **실제 방식** | `DOMParser`로 XHTML 파싱 → `<body>`의 `innerHTML`을 뷰어 컨테이너에 주입. `<style>`은 별도 주입. |
| **페이지 이동** | 뷰포트 높이 기준 **페이지 스냅**: ruler 복제 DOM에서 `getClientRects()`로 `lineBounds`를 만들고, 이전/다음 페이지에 맞는 줄 경계로 `scrollTop` 이동. |
| **터치** | `touchAction: pan-y`, 좌우 스와이프는 뷰어에서 처리. |

## 2. 위치 추적 (Locator)

### 2.1 CFI 제거

EPUB.js CFI 제거. 표준 locator:

```json
{ "chapterIndex": number, "blockIndex": number, "offset": number }
```

- **locatorUtils**: 정규화, 진도·북마크 페이로드, 코드포인트 오프셋(`[...str]` 기준) 등.

### 2.2 화면 가시 블록 vs 페이지 스냅

| 역할 | 방식 |
|------|------|
| **읽기 진도·노출 구간** | `IntersectionObserver`(컨테이너 root)로 가시 블록 수집 → 주기적으로 **가장 위·아래 블록**을 start/end로 하여 `startLocator`/`endLocator` emit. 블록 내 위치는 선택 영역이 있으면 `codePointOffsetInBlock` 반영. |
| **페이지 넘김·UI 페이지 인덱스** | **페이지 스냅** 기준. |

문서의 “IOB 기반 start/end”와 구현은 **동일 축**이다. 다만 진도 갱신은 **300ms 폴링**이므로 스크롤 직후 1프레임과 어긋날 수 있음 → 필요 시 간격·`requestAnimationFrame` 조합으로 조정 가능.

## 3. API v2

| 용도 | 엔드포인트 |
|------|------------|
| 진도 | `POST/GET/DELETE` `/api/v2/progress` |
| 매니페스트 | `GET /api/v2/books/{bookId}/manifest` |
| 북마크 | `/api/v2/bookmarks` (CRUD) |

클라이언트 캐시(`progressCache`, `manifestCache` 등)는 **서버의 `startLocator`/`endLocator` 스키마**와 1:1로 맞출 것.

## 4. 메타데이터(meta) 로딩

정규화 파이프라인의 **meta.json 동일 스키마**를 프론트가 쓴다 (`paragraphStarts`, `paragraphLengths`, `chapterIndex` 등).

| 순서 | 소스 |
|------|------|
| 1 | `GET /api/books/{bookId}/meta` → 성공 시 캐시(15분 TTL) |
| 2 | 실패 시 `public` **`{BASE_URL}books/{bookId}/meta.json`** fetch |

용도: **페이지 수 추정·챕터/블록 정합·스크롤 복원** 등. 진도의 txt 좌표 변환은 **백엔드**가 담당.

## 6. 백엔드·계약 리스크 (확인 필요)

| 이슈 | 설명 |
|------|------|
| **bookId vs jobId** | 일부 문서는 진도에 `jobId` 예시. 현재 프론트는 **bookId** 중심. 서버가 book↔job을 단일화했는지 확인. |
| **정규화 상태** | 도서 상세는 아직 `GET /api/books/{id}`(v1) 사용 가능. UI에 “정규화 완료/버전”이 필요하면 **`GET /api/v2/books/{id}`** 등 스펙 확정 후 연동. |
| **그래프** | `fine`/`macro`는 현재 `chapterIdx`·`eventIdx` 쿼리. locator 쿼리는 백엔드 스펙 후 확장. |

## 7. 로드맵 (우선순위)

### P0 — 계약·정합

- 진도 API: 식별자(bookId/jobId) 및 응답 locator 필드와 **캐시·페이로드 1:1 검증**.
- 재진입 시 locator 복원·챕터 경계 **수동 시나리오 테스트**.

### P1 — 기능·API

- 북마크 **locator 기반 Range 하이라이트** (`BookmarkManager` 보강).
- 도서 메타: v2 도서 API로 **정규화 상태·버전** UI 반영.
- 이벤트–블록: 서버가 `startBlockIndex`/`endBlockIndex`(또는 동등 정보) 제공 시 매칭 로직 강화.

### P2 — 확장·품질

- 그래프: locator 기반 쿼리·캐시 키 설계.
- 진도 IOB 폴링 최적화(간격, rAF).
- E2E·회귀(스와이프, 페이지 경계, 복원).
